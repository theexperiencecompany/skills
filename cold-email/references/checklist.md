# Pre-send checklist

Run every cold email and every sequence through this before it goes out.

## Per-email

- [ ] Subject is short, lowercase, specific/curious — not "quick question", "following up", no "!"
- [ ] Reads like a human typed it: no triads, no em-dash drama, no neat summarizing bows
- [ ] About *them* (outcome, time saved) more than about you (what you built)
- [ ] One concrete capability, not an abstract benefit ("drafts replies in your voice" > "AI-powered")
- [ ] Exactly **one** CTA
- [ ] Every link is real and resolves — no placeholders, no invented URLs
- [ ] CTA friction matches the trust level (reply/look early, buy late)
- [ ] ~5 short paragraphs or fewer
- [ ] Plain text for cold strangers (no logo header / hero / brand button)
- [ ] One-line unsubscribe present (legal requirement)
- [ ] Signed first-name only

## Per-sequence

- [ ] 4–5 touches, one every 2–3 days
- [ ] Shape: hook → value → proof → offer → close
- [ ] Email 1 has no hard ask (often no link) — it earns the next open
- [ ] The hard ask (offer) lands at touch 4, not touch 1
- [ ] Sequence leads with whatever is genuinely *news*
- [ ] Discount (if any) has a stated reason to exist + a real deadline or none
- [ ] Final email does NOT say "last one from me" unless it truly is
- [ ] Offer email includes the outcome math and an honest qualifier

## Deliverability gate (separate discipline, but it decides if copy is read)

- [ ] List is cleaned/verified — hard bounces and role accounts removed
- [ ] Sending from a subdomain, not the root domain that carries transactional mail
- [ ] Volume ramps (start small, most-recent/most-engaged first); bounces <~2%, complaints <~0.1%
- [ ] A/B the subject on a small slice before the full send
- [ ] Reply-to is a real inbox someone reads (replies are a positive signal)

## The two honesty checks

- [ ] If you claim personalization ("saw your post"), it's actually true for each recipient
- [ ] Every scarcity/deadline claim is real
